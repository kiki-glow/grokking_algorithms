# grokking_algorithms

## 1. Binary Search

with binary search, you guess the middle number and eliminate half the remaining numbers every time.
- for any list *n*, binary search will take *log₂ n* steps to run in the worst case, whereas simple/linear search will take *n* steps

## 2. Selection Sort

Select sort takes *O(n)* time, as you just saw. So you have an operation that takes O(n) time, and you have to do that *n* times:
- This takes **O(n × n)** time or **O(n²)** time
